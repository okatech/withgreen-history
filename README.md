## 1. プロジェクトの目的

- **公開発言だけで歴史を語る**：プレスリリース・インタビュー記事から武文社長本人の言葉を厳選し、二次解釈なしで提示する。
- **インタラクティブに体験できる**：単なる年表ではなく、横スクロール式タイムライン／店舗数推移グラフ／メニュー診断／ポイントカード解析など、スクロールするだけで楽しめる演出を多用。
- **ブランドへの敬意**：公式サイト（withgreen.club）のロゴ・コーポレートカラー（深いグリーン `#006323`）・タイポグラフィを尊重。
- **ファンの応援を形にする**：自分が一番通っている店舗を可視化し、「推し愛」を見える化する。

## 2. 技術スタック

| カテゴリ | 採用技術 |
|---|---|
| フロントエンド | **HTML5 / CSS3 / Vanilla JavaScript（ES2020+）** |
| ビルド | **不要**（単一HTMLファイル構成） |
| グラフ描画 | [Chart.js 4.x](https://www.chartjs.org/)（CDN） |
| OCR（日本語） | [Tesseract.js 5.1.0](https://tesseract.projectnaptha.com/)（CDN・遅延ロード） |
| OCR学習データ | tessdata `jpn`（CDN：`tessdata.projectnaptha.com/4.0.0_best`） |
| フォント | Google Fonts：Noto Sans JP / DM Sans |
| ホスティング | **GitHub Pages**（`master` ブランチのルート） |
| PPT生成（補助） | Python + `python-pptx`（`build_ppt.py`） |

設計方針：
- **依存ライブラリは最小限**。Chart.js／Tesseract.js は必要なときだけCDNから読み込む。
- **ビルド工程ゼロ**。`index.html` をブラウザで開けばそのまま動く。
- **オフライン優先（OCR以外）**：初回読み込み後は外部APIを叩かない。OCRも全処理がブラウザ内で完結（画像はサーバーに送信されない）。

## 5. 機能仕様

### 5.1 ヒーロー（KV）

- **左カラム**：英文ラベル（`A FOUNDER'S TEN-YEAR JOURNEY`）／和文大見出し／本文／バイライン（社長氏名・肩書）／CTAボタン2つ。
- **右カラム**：武文社長の **1:1 大判ポートレート（1920×1920）**。
  - 内側に白マット風フレーム
  - 左上に脈動アニメーション付き「FOUNDER · 2016 → 2026」タグ
  - 下部にグラデーション重ねの出典クレジット
- **背景演出**：薄いグリーンのブラシラインがふわっと回転。
- **クリック時のおまけ**：ヘッダー以外の任意の場所をクリックすると、**国産野菜の絵文字（🥬🥦🥕🍅🌽🥒…）** がパラパラと舞い落ちる演出（「日本の国産野菜だけ」のリクエストに対応）。

### 5.2 CHAPTER 01 / HISTORY：横スクロール・タイムライン

WithGreenの歴史を、武文社長本人の発言とともに **11枚の横並びカード** で表示。

- **データ**：`events[]` 配列（11オブジェクト、各 `{year, title, quote, src, url}`）。
- **UI**
  - カード幅：320px（モバイルは `86vw`）／カード間 18px
  - **CSS Scroll Snap**：`scroll-snap-type: x mandatory` でぴたっと止まる
  - **操作系**：
    - ◀ Prev / Next ▶ ボタン
    - ドラッグスクロール（マウスでも掴んで横移動）
    - 進捗バー＋現在番号カウンタ（例：`3 / 11`）
  - **コラプス機能**：各カードはデフォルトで本文5.1em分だけ表示（フェードアウト）。「+ 全文を読む」をクリックで全文展開、もう一度クリックで折りたたみ。
- **演出意図**：縦長で疲れる年表 → 横スクロールで「サクサク読み進める」体験に。長く感じさせない。

### 5.3 CHAPTER 02 / GROWTH：店舗数推移グラフ

- **ライブラリ**：Chart.js 4.x（折れ線＋エリア塗り）
- **データ**：年次ごとの店舗数（2016年=1店から2026年=50店まで）
- **演出**：
  - 緑のグラデーションエリア
  - 重要マイルストーン（10店達成・30店達成・50店達成）にアノテーションドット
  - 下に **3つのスタットカード**（創業年／達成年／総店舗数）

### 5.4 CHAPTER 03 / FOR YOU：心理テスト型サラダ診断

3問の質問に答えると、**WithGreenの公式メニューから1皿を提案**するパーソナル診断機能。

- **構造**
  - 各質問は3〜4の選択肢、各選択肢が `{ pts: { menuKey: weight, ... } }` の重みを持つ
  - 全質問終了時に最高スコアの `menuKey` を採用
  - 結果カード：メニュー画像／品名／キャッチ／タグ
- **メニューマスタ**（`wgMenus`）の例：
  ```js
  mushidori:   { name:'蒸し鶏のヘルシーボウル', img:'assets/salada-mushidori.png',  tags:['国産鶏','低糖質',...] }
  ebiavocado:  { name:'海老とアボカドのコブ風サラダ', ... }
  // ...全6種
  ```
- **修正済バグ**：
  - スマホで選択肢タップ後、次の質問の同位置ボタンが「ホバー残留」で選ばれた風に見えていた問題
  - 対策：`@media (hover: hover)` で `:hover` をマウス端末に限定／選択時に `blur()` ＋ `pointer-events:none` ＋ フェード遷移
- **再診断ボタン**：結果画面から「↻ もう一度診断する」で初期化可能
- **おまけ**：結果が出ると `saladRain(15, 2200)` で野菜絵文字がお祝いの雨

### 5.5 CHAPTER 04 / FAN POWER：ポイントカード解析

**手持ちのWithGreenポイントカード写真をアップロードすると、スタンプの店舗名をAI（Tesseract.js）で読み取り、「あなたの推し店舗」を可視化する**機能。

#### 5.5.1 入力

| 方法 | 説明 |
|---|---|
| 📷 写真を選ぶ | ファイルピッカー（JPEG/PNG/HEIC、〜10MB目安） |
| ドラッグ＆ドロップ | アップロードゾーンへ画像をドロップ |
| 🎲 デモを試す | サンプルデータで即座に結果体験（OCR不要） |

#### 5.5.2 解析パイプライン

```
[画像] → ① 前処理 → ② OCR(4回転) → ③ テキスト統合 → ④ 店舗マッチング → ⑤ 可視化
```

1. **前処理（`preprocessImage`）**
   - 画像を Canvas に展開
   - 長辺 1800px までアップスケール（小さなスタンプ文字の可読性向上）
   - グレースケール化（`0.299R + 0.587G + 0.114B`）
   - 平均輝度ベースの **適応的しきい値** でソフト2値化（コントラスト×1.7、暗部→真黒・明部→真白・中間はグラデ保持）
2. **多角度OCR**
   - 0° / 90° / 180° / 270° の **4方向** で個別にOCR実行
   - 手押しスタンプは向きがバラバラなので、必ずどこかの回転で読める可能性を上げる
   - Tesseract.js v5：`createWorker('jpn', 1)` ＋ `tessedit_pageseg_mode='11'`（**SPARSE_TEXT** モード＝散在テキスト）
3. **店舗マッチング（`matchStoresFromText`）**
   - 全方向のOCR結果を結合し、ノイズ文字（句読点・括弧等）を除去
   - 公式店舗マスタ（`wgStores`、40+ 店舗）の各 **キーワード配列** に対し2段階マッチング：
     - **(a) 完全一致**：キーワードが何回出現するかをカウント（複数スタンプ＝複数ヒットを想定）
     - **(b) 部分一致**：4文字以上のキーワードに限り、1文字抜け／置換に耐えるサブシーケンス検索（OCR誤認識への耐性）
   - ヒット数 = そのお店のスタンプ数として採用

#### 5.5.3 出力

- **YOUR FAVORITE No.1 ヒーローカード**
  - 王冠絵文字 👑（ふわふわ揺れるアニメーション）
  - グリーンのグラデーション背景＋光の演出
  - 店舗名と、訪問回数に応じた応援メッセージ（5段階で変化）
- **横棒グラフ**
  - ランキング表示（金🥇／銀🥈／銅🥉のメダル風）
  - ハート ♥ の数で訪問回数を視覚化
  - 棒のアニメーション（`cubic-bezier(.2,.9,.3,1)` でスルッと伸びる）
- **編集UI**
  - 各店舗の `+` / `−` / `×` ボタンで手動補正
  - 「店舗を追加…」ドロップダウンから40+店舗を選んで追加
- **🔍 OCR読取結果（デバッグ）**
  - `<details>` 要素で折りたたみ可能
  - AIが読み取った生テキストをそのまま表示し、なぜマッチしたか／なぜマッチしなかったかをユーザー自身が検証できる

#### 5.5.4 設計上の工夫

| 課題 | 対策 |
|---|---|
| 手押しスタンプは低コントラスト・回転・かすれ | 前処理で明るさ正規化＋2値化／4角度OCR |
| OCRの完璧さは期待できない | 部分一致マッチング＋手動補正UIを最初から提供 |
| OCR学習データのDLが重い（数MB） | 初回利用時のみ・遅延ロード（CDN） |
| プライバシー | **画像は一切サーバーに送られない**（クライアント完結）旨をUIに明記 |
| ライブラリ実行失敗 | try-catch でエラーメッセージ表示／手動入力モードに自動移行 |

#### 5.5.5 店舗マスタの構造

```js
const wgStores = [
  { n: '日本橋高島屋S.C.店',           kw: ['日本橋', '高島屋'] },
  { n: '東京ミッドタウン八重洲店',     kw: ['八重洲', 'ミッドタウン'] },
  { n: '銀座ノボ店',                   kw: ['銀座', 'ノボ'] },
  // ... 全40+店舗
];
```

`n` = 表示名／`kw` = OCR文字列とマッチさせるキーワード配列。1店舗に複数キーワードを持たせることで、「どっちの単語が読めても拾える」設計。

## 9. プライバシー

- **アップロードされたポイントカード画像は、一切サーバーに送信されません。**
- OCR処理（Tesseract.js）はすべてブラウザ内で実行されます。
- 唯一の外部通信は以下の **静的アセット取得** のみ：
  - Chart.js（jsdelivr CDN）
  - Tesseract.js + jpn 学習データ（jsdelivr / projectnaptha CDN）
  - Google Fonts
- アクセス解析・トラッキングは導入していません。

## 10. セットアップ・開発

### 10.1 ローカルで動かす

```bash
# クローン
git clone https://github.com/okatech/withgreen-history.git
cd withgreen-history

# 任意の静的サーバーで配信（例：Python標準）
python -m http.server 8080
# → http://localhost:8080/ をブラウザで開く
```

> ※ `index.html` をダブルクリックして `file://` で開いても基本動作しますが、
> Tesseract.js のWebWorker読込やfetchの一部が CORS で失敗することがあるため、ローカルサーバー経由での起動を推奨します。

### 10.2 PPT資料の再生成

```bash
pip install python-pptx pillow
python build_ppt.py
# → WithGreen_History.pptx が更新される
```

### 10.3 デプロイ

`master` ブランチへのpushで GitHub Pages が自動デプロイされます（`Settings > Pages > Source: master / root`）。
